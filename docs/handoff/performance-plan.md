# 性能排查与优化计划

## 当前慢点判断

当前慢点主要不在单纯数据库读写，而在以下链路：

- 高中或更高难度题干改写最慢，主因是模型调用
- 四个选项中文释义补齐会额外触发模型调用
- 飞书整表读取会放大延迟
- 复习链查询会放大延迟

## Supabase 的作用边界

Supabase 可以改善读写、查询和统计，但不能直接解决模型调用慢的问题。

因此，数据库迁移不能被视为全部性能问题的唯一解法。

## 建议优化顺序

1. 先加计时日志
2. 再缓存中文释义
3. 再减少非必要模型调用
4. 再评估飞书表读取和复习链查询优化
5. 最后确认 Supabase 是否需要进入迁移方案

## 已观察到的性能相关位置

- `app/backend/feishu.js`：题目生成、中文释义补齐、MiniMax 调用、飞书表读写
- `app/backend/language-enrichment.js`：题干/释义重写
- `app/backend/option-meanings.js`：选项中文释义补齐
- `app/backend/review-service.js`：复习链和缓存
